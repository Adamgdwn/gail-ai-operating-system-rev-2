# Roadmap

## Now

- box the current coordination pivot across GAIL AI Operating System Rev 2,
  Freedom, and AG Operations Workspace
- let AG Operations Workspace finish its current evolution before running a
  consolidation review
- keep Rev 2's default implementation runway at Chunk Twenty, but resume it
  only after acknowledging the three-repo direction

## Next

- use the build consolidation decision process to decide whether the three
  builds should stay separate, bridge, fold, or defer
- if Rev 2 implementation resumes, add local governed approval actions without
  live tool execution, shared runtime, or connector activation
- improve documentation and operating safety where the active build path needs
  it most

## Later

- add growth features only after ownership boundaries are stable
- revisit architecture and governance controls after the three-repo decision
  process has owner acceptance
